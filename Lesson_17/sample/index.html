§01

Text & Headings

The tags that carry meaning and hierarchy in your content — search engines and screen readers both read this structure.

<h1>–<h6>

Heading 1

Heading 2

Heading 3

Heading 4

<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>

Only one <h1> per page. Levels should nest in order — don't skip from h2 to h4.

<p>

A paragraph is a block of running text — the default container for everything you write. 

<p>A paragraph of text.</p>

The most common tag on the web. Browsers add space above and below it automatically.

<strong> <em>

This step is required, but the order is up to you. 

<strong>required</strong>
<em>up to you</em>

strong = importance (bold). em = emphasis (italic). Not just visual — screen readers announce both differently.

<blockquote>
Simplicity is the soul of efficiency.
<blockquote>
  Simplicity is the soul
  of efficiency.
</blockquote>

For quoting an external source — pairs well with a <cite>.

<hr>

Section one.

Section two.

<p>Section one.</p>
<hr>
<p>Section two.</p>

Marks a thematic break — a topic change, not just a decorative line.

§02

Lists

Three ways to group related items — pick the one that matches the meaning.

<ul>
  • HTML — structure
  • CSS — style
  • JavaScript — behavior
<ul>
  <li>HTML</li>
  <li>CSS</li>
</ul>

Unordered list — the sequence doesn't matter.

<ol>
  1. Plan the page
  2. Write the HTML
  3. Style it with CSS
<ol>
  <li>Plan the page</li>
  <li>Write the HTML</li>
</ol>

Ordered list — steps, rankings, anything where order carries information.

<dl>
HTML
Defines structure and content.
CSS
Defines presentation and layout.
<dl>
  <dt>HTML</dt>
  <dd>Structure.</dd>
</dl>

Description list — pairs a term (dt) with its definition (dd). Great for glossaries and specs.

§03

Links & Media

How pages connect to each other, and how images get labeled.

<a>

Visit MDN Web Docs 

<a href="https://mdn.io"
   target="_blank"
   rel="noopener">
  Visit MDN Web Docs
</a>

href points where it goes. target="_blank" opens a new tab — pair it with rel="noopener" for security.

<figure> 
<figure>
  <img src="photo.jpg"
       alt="What it shows">
  <figcaption>Fig. 1</figcaption>
</figure>

alt is not optional — it's what screen readers announce, and what shows if the image fails to load.

§04

Tables

For genuinely tabular data — rows and columns of related values. Not for page layout.

<table>
Course Enrollment
Student Language Status
Amina Python Active
Lucas Flask Active
Yara JavaScript Waitlist 
<table>
  <caption>Course Enrollment</caption>
  <thead>
    <tr><th>Student</th><th>Language</th></tr>
  </thead>
  <tbody>
    <tr><td>Amina</td><td>Python</td></tr>
  </tbody>
</table>

thead/tbody group rows semantically. th marks a header cell — use scope="col" or scope="row" for accessibility.

§05

Forms

How a page collects input. Every field needs a <label> — it's not decoration, it's what makes the field usable.

<form>
Contact
Preferred contact 
<form>
  <label for="email">Email
    <input type="email" id="email">
  </label>
  <button type="submit">Send</button>
</form>

This demo form doesn't submit anywhere — the button is type="button" on purpose. In a real form, use type="submit" with an action and method.

§06

Semantic Layout

Landmarks that describe the role of a region, not just its box. This whole page is built from these.

<header> <nav> <main> <footer>
header
nav
section
article
aside
footer
<header>…</header>
<nav>…</nav>
<main>
  <section>
    <article>…</article>
  </section>
  <aside>…</aside>
</main>
<footer>…</footer>

main appears once per page (this page's real <main> wraps everything below the sidebar). article is self-contained content; section groups related content; aside is tangential.

§07

div & span

When nothing more meaningful fits — pure boxes for layout and styling hooks.

<div>
A block-level box with no meaning of its own — a hook for CSS. 
<div class="card">
  content
</div>

Reach for a semantic tag first (§06). Use div when none of them fit.

<span>

This sentence has one highlighted word using a span. 

one <span class="hl">
  highlighted
</span> word

Same idea as div, but inline — it doesn't start a new line.

§08

CSS Concepts

The properties that turn plain HTML into a designed page. Full commented examples live in style.css.

selectors

Element selector → matches every <p>

Class selector → matches .highlight

ID selector → matches #unique, once per page

p          { color: navy; }
.highlight { background: yellow; }
#unique    { font-weight: bold; }

Classes are reusable. IDs are unique. Prefer classes for styling.

box model
margin
border
padding
content
.box {
  margin: 20px;
  border: 4px solid;
  padding: 16px;
}

Every element is a box, from the inside out: content → padding → border → margin.

color / font

font-family, font-weight, font-size

color: sets the text color

background-color: sets the fill

.text {
  color: #c0392b;
  font-family: monospace;
  font-weight: 700;
}

The most-used properties in any stylesheet.

display: flex
1
2
3
.row {
  display: flex;
  gap: 12px;
  justify-content: space-between;
}

One-dimensional layout — arranges items in a row or column. This whole page uses flex for its layout.

display: grid
1
2
3
4
5
6
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 12px;
}

Two-dimensional layout — rows and columns at once. The demo-grids on this page use it too.

position
absolute

The parent is position: relative; the badge is position: absolute, placed relative to it. 

.parent { position: relative; }
.badge  {
  position: absolute;
  top: 10px; right: 10px;
}

absolute positions relative to the nearest positioned ancestor — not the whole page.

:hover / :focus-visible 
.btn:hover {
  background: white;
}
.btn:focus-visible {
  outline: 3px solid orange;
}

Pseudo-classes style an element based on state — no JavaScript required.

transition 
.box {
  transition: transform 0.35s ease;
}
.box:hover {
  transform: rotate(45deg) scale(1.15);
}

Animates a property change smoothly instead of snapping instantly. Hover the square.

@media 
@media (max-width: 700px) {
  .bar { background: coral; }
}

Resize this browser window below 700px — the bar above reacts on its own. This is how the sidebar on this page collapses on mobile, too.
